Section 01

Why Linear Regression Doesn't Work for Classification

So far, Module 2 and Module 2.1 dealt with regression — predicting a continuous number like house price. Now consider a different kind of problem:

🎯 A New Kind of Question

Given a tumour's size, is it malignant (1) or benign (0)? Given an email's content, is it spam (1) or not spam (0)? Given a student's exam scores, will they pass (1) or fail (0)?

These are classification problems — the output y can only take a small number of discrete values. The simplest case, which we focus on in this module, is binary classification: y ∈ {0, 1}.

SymbolNameMeaning
yLabel / classThe output, but now restricted to y = 0 or y = 1
y = 0Negative class"No", "Benign", "Not spam" — whatever we call the negative outcome
y = 1Positive class"Yes", "Malignant", "Spam" — the outcome we are often trying to detect
ŷPredictionNow interpreted as a probability that y = 1, not a raw number

What Happens If We Just Use Linear Regression Anyway?

Let's try it. Suppose we plot tumour size (x) vs. malignant (y=1) / benign (y=0), and fit a straight line ŷ = wx + b as in Module 2:

⚠️ Two Big Problems With Linear Regression Here

1. Predictions go outside [0, 1]. ŷ = wx + b can output values like −2 or 5.7 — but a probability must be between 0 and 1. These values are meaningless as probabilities.

2. Extremely sensitive to outliers. As shown in Figure 1, one new data point far to the right can drag the entire line and shift the decision boundary — even though that point doesn't change the underlying pattern at all.

We need a model whose output is always between 0 and 1, so it can be interpreted as a probability, and which doesn't shift wildly with outliers. This is exactly what logistic regression provides.

Section 02

The Sigmoid Function — Squashing Numbers into [0, 1]

Recall the linear regression model from Module 2.1: z = wᵀx + b. This z can be any real number — from −∞ to +∞. We need a function that takes z and "squashes" it into the range [0, 1]. That function is the sigmoid function (also called the logistic function — hence the name "logistic regression").

The Sigmoid Function
g(z) = 1 / (1 + e−z)
e ≈ 2.718 is Euler's number. As z ranges over all real numbers, g(z) always lies between 0 and 1.

What Does the Sigmoid Curve Look Like?

The Logistic Regression Model

We build the logistic regression model in two steps — exactly the same first step as Module 2.1, then a new second step:

1

Compute z — same as Module 2.1

Compute the familiar linear combination of features:

z = wᵀx + b
2

Apply the sigmoid function to z

Pass z through g(·) to squash it into [0, 1]:

ŷ = g(z) = g(wᵀx + b) = 1 / (1 + e−(wᵀx+b))
Logistic Regression Model
ŷ = 1 / (1 + e−(wᵀx+b))
Same parameters w and b as Module 2.1 — but the output is now squashed into [0, 1] and interpreted as a probability.

Interpreting the Output as a Probability

The output ŷ is interpreted as the probability that y = 1, given input x:

Probabilistic interpretation
ŷ = P(y=1 | x; w, b)
"The probability that y=1, given input x, parameterised by w and b"
💡 Concrete Example

Suppose x = tumour size, and after computing ŷ we get ŷ = 0.7. This means: "There is a 70% chance this tumour is malignant (y=1)". Since probabilities must sum to 1: P(y=0|x) + P(y=1|x) = 1, so P(y=0|x) = 0.3 — a 30% chance it's benign.

Section 03

The Decision Boundary — Turning Probability into a Class

The model ŷ outputs a probability — a number between 0 and 1. But ultimately, we want to make a decision: is this tumour malignant or not? We need a rule to convert the probability ŷ into a final class prediction, call it class ∈ {0, 1}.

The 0.5 Threshold Rule

The most common rule: predict class = 1 if ŷ ≥ 0.5, otherwise predict class = 0.

Default classification rule
class = 1  if  ŷ ≥ 0.5     class = 0  if  ŷ < 0.5

Recall from Figure 2: g(z) ≥ 0.5 exactly when z ≥ 0. So the threshold ŷ ≥ 0.5 translates directly into:

Equivalent condition on z
class = 1  if  wᵀx + b ≥ 0     class = 0  if  wᵀx + b < 0
The decision boundary is the line (or curve) where wᵀx + b = 0 exactly.

Visualising the Decision Boundary — Linear Case

Suppose we have two features x₁ and x₂, and after training we get w₁ = 1, w₂ = 1, b = −3. The decision boundary is the line x₁ + x₂ = 3:

Non-Linear Decision Boundaries

What if the two classes can't be separated by a straight line? Just like Module 2.1's polynomial regression, we can add polynomial features like x₁², x₂², x₁x₂ to z = wᵀx + b. The decision boundary z = 0 then becomes a curve (e.g. a circle or ellipse):

✅ Reusing Module 2.1's Big Idea

Just as Module 2.1 showed that x², x³, √x are "just new features" for linear regression, the same trick works here: x₁², x₂², x₁x₂ are new features for logistic regression. The algorithm doesn't change — only the feature engineering does. This lets logistic regression model arbitrarily complex decision boundaries.

Section 04

The Cost Function — Why Squared Error Fails Here

What Goes Wrong If We Reuse Module 2's Cost Function?

In Module 2, we used J(w,b) = (1/2m) Σ (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾)² — the squared error cost. What if we plug our new sigmoid-based ŷ into this same formula?

⚠️ The Problem — A Non-Convex, Wiggly Cost Surface

Because ŷ = g(wᵀx+b) involves the sigmoid (a nonlinear function), squaring (ŷ−y) and summing produces a cost function J(w,b) that is non-convex — it has many local minima ("wiggles") like an egg carton. Gradient descent could get stuck in a local minimum and never find the true best w, b.

The Solution — The Logistic Loss Function

Instead of squared error, logistic regression uses a different per-example loss function L, designed specifically to remain convex when combined with the sigmoid:

Logistic Loss for one example (i)
L(ŷ⁽ⁱ⁾, y⁽ⁱ⁾) = −log(ŷ⁽ⁱ⁾)    if y⁽ⁱ⁾ = 1 L(ŷ⁽ⁱ⁾, y⁽ⁱ⁾) = −log(1 − ŷ⁽ⁱ⁾)    if y⁽ⁱ⁾ = 0

Why Does This Make Intuitive Sense?

Let's break down each case and see why this loss function "punishes" wrong predictions heavily and "rewards" correct ones.

💡 Intuition Summary

If y=1 (the true answer is "yes") and the model predicts ŷ close to 1 (also "yes, confidently") → loss ≈ 0. Good!
If y=1 but the model predicts ŷ close to 0 (confidently "no") → loss → ∞. The model is severely punished for being confidently wrong.

The same logic applies symmetrically when y=0.

Combining Both Cases into One Formula

Writing two separate cases is correct but inconvenient for taking derivatives. Since y is always either 0 or 1, we can combine both cases into a single elegant formula:

Combined Logistic Loss — single formula
L(ŷ⁽ⁱ⁾, y⁽ⁱ⁾) = −y⁽ⁱ⁾log(ŷ⁽ⁱ⁾) − (1−y⁽ⁱ⁾)log(1−ŷ⁽ⁱ⁾)
When y⁽ⁱ⁾=1, the second term vanishes (multiplied by 0) and we get −log(f). When y⁽ⁱ⁾=0, the first term vanishes and we get −log(1−f). Same formula, both cases!

Walking Through All Four Possibilities

For simplicity, drop the superscript ⁽ⁱ⁾, so the combined formula reads L = −y·log(ŷ) − (1−y)·log(1−ŷ). There are exactly four combinations of (true label y, prediction ŷ) — let's check that the loss behaves correctly in every single one.

Case Reasoning Result
y=1
ŷ=1		 Since y=1, the term −(1−y)·log(1−ŷ) becomes −(1−1)·log(…) = −0·log(…) = 0. It vanishes.
The remaining term is −y·log(ŷ) = −1·log(1).
Since log(1) = 0, this gives −1·0 = 0. 		L = 0
✓ Correct, confident prediction — no penalty
y=1
ŷ=0		 Since y=1, again −(1−y)·log(1−ŷ) = −0·log(…) = 0. It vanishes.
The remaining term is −y·log(ŷ) = −1·log(0).
Since log(0) = −∞, this gives −1·(−∞) = +∞. 		L → ∞
✗ Wrong, confident prediction — huge penalty
y=0
ŷ=0		 Since y=0, the term −y·log(ŷ) becomes −0·log(…) = 0. It vanishes.
The remaining term is −(1−y)·log(1−ŷ) = −(1−0)·log(1−0) = −1·log(1).
Since log(1) = 0, this gives −1·0 = 0. 		L = 0
✓ Correct, confident prediction — no penalty
y=0
ŷ=1		 Since y=0, again −y·log(ŷ) = −0·log(…) = 0. It vanishes.
The remaining term is −(1−y)·log(1−ŷ) = −(1−0)·log(1−1) = −1·log(0).
Since log(0) = −∞, this gives −1·(−∞) = +∞. 		L → ∞
✗ Wrong, confident prediction — huge penalty
✅ The pattern in one sentence

In every row, exactly one of the two terms in −y·log(ŷ) − (1−y)·log(1−ŷ) survives (the other multiplies by 0 and vanishes). The surviving term is 0 when the prediction matches the true label (because log(1)=0), and shoots to +∞ when the prediction confidently contradicts the true label (because log(0)=−∞). This single formula automatically handles all four cases correctly — that's the elegance of combining them.

The Full Cost Function J(w, b)

Exactly as in Module 2, the cost function J is the average loss over all m training examples:

Logistic Regression Cost Function
J(w,b) = 1/m ∑i=1m L(ŷ⁽ⁱ⁾, y⁽ⁱ⁾)
= −1/m ∑i=1m [y⁽ⁱ⁾log(ŷ⁽ⁱ⁾) + (1−y⁽ⁱ⁾)log(1−ŷ⁽ⁱ⁾)]
This is called the binary cross-entropy loss. Unlike squared error + sigmoid, this J(w,b) IS convex — gradient descent is guaranteed to reach the global minimum.

Module 2 — squared error

J(w,b) = (1/2m)Σ(f(x⁽ⁱ⁾)−y⁽ⁱ⁾)². Perfect for regression — convex when f is linear. Non-convex when f involves sigmoid.

Module 3 — logistic loss

J(w,b) = −(1/m)Σ[y log(f) + (1−y)log(1−f)]. Designed specifically so J stays convex even with the sigmoid in f.

Section 05

Gradient Descent for Logistic Regression

The Astonishing Result

We derived a brand new cost function J(w,b) — the binary cross-entropy. You might expect the gradient descent update rules to look completely different from Module 2. They don't.

Gradient Descent — Logistic Regression (repeat until convergence)
wj := wj − α · 1/m ∑i=1m (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾) xj⁽ⁱ⁾ b   := b   − α · 1/m ∑i=1m (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾)
Compare to Module 2.1's update rules — they look identical! The only difference is what ŷ means.

Side-by-Side: Module 2.1 vs Module 3

Module 2.1 (Linear Regression) Module 3 (Logistic Regression)
prediction ŷ = wᵀx + b ŷ = g(wᵀx + b)   — sigmoid!
J(w,b) (1/2m)Σ(ŷ−y)² −(1/m)Σ[y log ŷ + (1−y)log(1−ŷ)]
wⱼ update wⱼ := wⱼ − α(1/m)Σ(ŷ−y)xⱼ wⱼ := wⱼ − α(1/m)Σ(ŷ−y)xⱼ   ← same form!
b update b := b − α(1/m)Σ(ŷ−y) b := b − α(1/m)Σ(ŷ−y)   ← same form!
✅ Why This Matters

The update rule formula looks identical to Module 2.1's. The crucial difference is hidden inside ŷ: in Module 2.1, ŷ = wᵀx+b is a straight line; in Module 3, ŷ = g(wᵀx+b) involves the sigmoid. Same update rule structure, different underlying function — this is why we kept the exact same notation. Everything you learned about gradient descent, learning rate α, convergence checking, and feature scaling from Module 2.1 applies directly here, unchanged.

The Algorithm Step by Step

1

Forward pass — compute z and then ŷ

For every example i: compute z⁽ⁱ⁾ = wᵀx⁽ⁱ⁾ + b, then ŷ⁽ⁱ⁾ = g(z⁽ⁱ⁾) = 1/(1+e−z⁽ⁱ⁾).

2

Compute the error f⁽ⁱ⁾ − y⁽ⁱ⁾

This looks exactly like Module 2's error ŷ−y, except f⁽ⁱ⁾ is now a probability between 0 and 1, and y⁽ⁱ⁾ is exactly 0 or 1.

3

Compute gradients ∂J/∂wⱼ and ∂J/∂b

Average (f⁽ⁱ⁾−y⁽ⁱ⁾)·xⱼ⁽ⁱ⁾ for each weight, and average (f⁽ⁱ⁾−y⁽ⁱ⁾) for the bias — identical formula structure to Module 2.1.

4

Update wⱼ and b simultaneously

wⱼ := wⱼ − α·∂J/∂wⱼ   and   b := b − α·∂J/∂b, using old values for both — the same simultaneous update rule from Module 2.

Section 07

Overfitting & Regularisation — A Preview

The Problem with Too Many Features

Module 2.1 showed that adding polynomial features (x², x³, √x) can help fit curved data. Module 3 showed the same trick creates curved decision boundaries. But what happens if we add too many features, or features of very high degree?

⚠️ The Overfitting Problem

An overfit model achieves J(w,b) ≈ 0 on the training set — it appears to "perfectly" fit the data. But when shown new examples it has never seen, it performs poorly. The model has memorised the training set rather than learning the underlying pattern.

Regularisation — A Quick Preview

The fix (covered in detail in the next module) is regularisation: add a penalty term to J(w,b) that discourages the weights wⱼ from becoming too large. Intuitively, large weights are what allow the model to create wildly wiggly boundaries — penalising them keeps the boundary smooth.

Preview — Regularised cost function
J(w,b) = [original cost]  +  λ/2m ∑j=1n wj²
λ (lambda) is a new hyperparameter controlling how much we penalise large weights. λ=0 means no regularisation (risk of overfitting); very large λ means heavily penalised weights (risk of underfitting). Covered fully in Module 4.

Practice Questions

Think & Answer

Work through these questions on your own before revealing the hints. They cover the sigmoid function, logistic loss, decision boundaries, and gradient descent — exactly the kinds of questions you'd see in an exam on logistic regression.

Easy Medium Hard
Part A — Conceptual & Intuition
Q1 Why can't we use linear regression for a binary classification problem like spam detection? What goes wrong?
Easy Concept
▾
Sample Answer

Linear regression outputs any real number (e.g. −3, 0.7, 5.2), but a binary classifier must output a value between 0 and 1 to represent a probability. If you force a threshold like 0.5 onto a linear output, adding more extreme data points can shift the line and move the threshold to the wrong place, misclassifying examples that were previously correct. Logistic regression solves this by squashing the output through the sigmoid function, guaranteeing an output in (0, 1).

Q2 Describe the sigmoid function in words. What are its output range and its value at z = 0?
Easy Concept
▾
Sample Answer

The sigmoid function g(z) = 1 / (1 + e⁻ᶻ) takes any real number z and squashes it into the range (0, 1). As z → +∞, g(z) → 1; as z → −∞, g(z) → 0. At z = 0, g(0) = 1/(1+1) = 0.5 exactly. This means the model is 50% confident when z = 0 — that is the decision boundary.

Q3 What is a decision boundary? For a logistic regression model, where exactly does it lie?
Easy Concept
▾
Sample Answer

The decision boundary is the line (or surface) that separates the region where the model predicts class 1 from the region where it predicts class 0. For logistic regression with a threshold of 0.5, this corresponds to g(z) = 0.5, which occurs when z = 0. So the decision boundary is defined by the equation w·x + b = 0 — a line in 2D, a plane in 3D, a hyperplane in higher dimensions.

Q4 Why don't we use the mean squared error (MSE) cost function for logistic regression? What problem does it cause?
Medium Concept
▾
Sample Answer

When the sigmoid function is combined with MSE, the resulting cost surface becomes non-convex — it has many local minima. Gradient descent can get stuck in one of these local minima and never find the globally best parameters.

The logistic (cross-entropy) loss is carefully chosen so that when composed with the sigmoid, the cost surface is convex — shaped like a bowl with a single global minimum — guaranteeing that gradient descent will always converge to the best solution.

Q5 Explain the intuition behind the logistic loss for a single example. Why does the loss become very large when the model is confidently wrong?
Medium Concept
▾
Sample Answer

When y = 1, the loss is −log(f). If the model predicts f ≈ 1 (correctly confident), then −log(1) = 0 — no penalty. But if the model predicts f ≈ 0 (confidently wrong), then −log(0) → ∞ — a huge penalty. The same logic applies when y = 0 using −log(1 − f).

This asymmetry is exactly what we want: the algorithm is punished severely for being very sure about the wrong answer, forcing it to quickly move the parameters away from such bad regions.

Part B — Calculation & Application
Q6 A logistic regression model has w = [2, −1] and b = −3. A test example is x = [1, 2]. Compute z, then g(z) (the predicted probability). Would this example be classified as 1 or 0 using a 0.5 threshold?
Easy Math
▾
Sample Answer

Step 1 — Compute z:

z = w·x + b = (2)(1) + (−1)(2) + (−3) = 2 − 2 − 3 = −3

Step 2 — Apply sigmoid:

g(−3) = 1 / (1 + e³) ≈ 1 / (1 + 20.09) ≈ 0.047

The predicted probability is ≈ 4.7%, which is well below 0.5, so the example is classified as class 0.

Q7 Compute the logistic loss for a single example where y = 1 and the model predicts f = 0.8. Then compute it for f = 0.2. Which loss is larger and why?
Easy Math
▾
Sample Answer

When y = 1, the loss is L = −log(f).

L(f = 0.8) = −log(0.8) ≈ 0.223 L(f = 0.2) = −log(0.2) ≈ 1.609

The loss for f = 0.2 is much larger because the model is predicting only 20% probability for a true positive — it is confidently wrong. The logarithm penalises this heavily, which is exactly the desired behaviour.

Q8 Write the gradient descent update rule for w₁ in logistic regression. How does it look compared to the linear regression update rule? Are they the same or different?
Medium Math
▾
Sample Answer
w₁ := w₁ − α · (1/m) Σᵢ (f⁽ⁱ⁾ − y⁽ⁱ⁾) · x₁⁽ⁱ⁾

The formula looks identical to linear regression! The key difference is what f⁽ⁱ⁾ means: in linear regression f⁽ⁱ⁾ = w·x + b (raw linear output), while in logistic regression f⁽ⁱ⁾ = g(w·x + b) (sigmoid-transformed output). The shape of the update rule is the same; only the forward pass differs.

Q9 A model has w = [3, −2] and b = 1 with two features. Write the equation of its decision boundary and describe what it looks like geometrically in 2D.
Medium Math
▾
Sample Answer

The decision boundary occurs where z = 0:

3x₁ − 2x₂ + 1 = 0

Rearranging: x₂ = (3x₁ + 1) / 2. This is a straight line in the (x₁, x₂) plane. Points above the line (where z < 0) will be predicted as class 0; points below (where z > 0) will be predicted as class 1. Logistic regression always produces a linear decision boundary with standard features.

Q10 Given a single training example x⁽¹⁾ = [1, 1], y⁽¹⁾ = 1, and current parameters w = [0, 0], b = 0 with α = 0.1, perform one full gradient descent step for w₁, w₂, and b. (Use the logistic regression update rule.)
Hard Math
▾
Sample Answer

Step 1 — Forward pass:

z = 0(1) + 0(1) + 0 = 0 → f = g(0) = 0.5

Step 2 — Error (m = 1):

f − y = 0.5 − 1 = −0.5

Step 3 — Gradients:

∂J/∂w₁ = (−0.5)(1) = −0.5 ∂J/∂w₂ = (−0.5)(1) = −0.5 ∂J/∂b = −0.5

Step 4 — Updates:

w₁ := 0 − 0.1(−0.5) = 0.05 w₂ := 0 − 0.1(−0.5) = 0.05 b := 0 − 0.1(−0.5) = 0.05

All parameters increase, which makes sense: the model predicted 0.5 when the true label was 1, so it needs to increase z (and therefore f) by increasing the weights.
