📌 Note on x⁽ⁱ⁾

In Module 2, x⁽ⁱ⁾ was a single number. Now it is a vector of n numbers — one per feature. For example, x⁽¹⁾ = [2104, 5, 1, 45]. The superscript ⁽ⁱ⁾ still means "i-th example" — nothing has changed there.

The New Model — One Term Per Feature

In Module 2, the model was ŷ = wx + b (one weight). With n features, we simply add one weight per feature:

Multiple Linear Regression Model
ŷ = w₁x₁ + w₂x₂ + w₃x₃ + … + wₙxₙ + b
Each feature xⱼ gets its own weight wⱼ. The bias b is still a single number.

For our house-price example with n = 4:

Concretely for n = 4 features
ŷ = w₁·(size) + w₂·(bedrooms) + w₃·(floors) + w₄·(age) + b
For instance, w₁ = 0.1 means: each extra sq ft adds $100 to the predicted price.
💡 Intuition

Think of each wⱼ as the "importance weight" of feature j. A large positive w₁ means bigger houses cost more. A large negative w₄ means older houses cost less. The algorithm's job is to find the right combination of all the w's and b.

In that case we would compute the predictions for all examples at once using matrix multiplication. But for the vectors w and x matrix multiplication would not work.

Because, both w and x are column vectors of length n (n rows, 1 column). If we try to multiply them as-is, matrix multiplication rules say we need the inner dimensions to match. In that case if we take the transpose of w, we get a row vector, and the multiplication would work.

❌ w · x — doesn't work

w is (n × 1) and x is (n × 1). Matrix multiplication (n×1)·(n×1) is undefined — inner dimensions n and n don't match in the right way for a dot product to give a scalar.

✅ wᵀ · x — works perfectly

wᵀ is (1 × n) and x is (n × 1). Multiplication (1×n)·(n×1) = (1×1) = a scalar — exactly one predicted price. The inner dimensions match!

Final compact form — Andrew Ng notation

wᵀ is the transpose of w. This is the standard notation Andrew Ng uses in his course.

Section 03

Updated Gradient Descent for n Features

In Module 2, gradient descent updated just two parameters: w and b. Now with n features, we have n + 1 parameters to update: w₁, w₂, …, wₙ and b. The good news is the structure is identical — we just repeat the update for each wⱼ.

The Cost Function — Same Formula, More Parameters

The cost function J does not change its shape — it still measures average squared error. But now ŷ⁽ⁱ⁾ depends on all features:

Cost function for multiple regression
J(w, b) = 1/2m ∑i=1m (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾)²
Where ŷ⁽ⁱ⁾ = w₁x₁⁽ⁱ⁾ + w₂x₂⁽ⁱ⁾ + … + wₙxₙ⁽ⁱ⁾ + b = wᵀx⁽ⁱ⁾ + b

The Updated Update Rules

In Module 2 we had just one update for w. Now we have one update rule for each wⱼ, plus the same update for b:

Gradient Descent — Multiple Features (repeat until convergence)
w₁ := w₁ − α · 1/m ∑i=1m (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾) x₁⁽ⁱ⁾
w₂ := w₂ − α · 1/m ∑i=1m (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾) x₂⁽ⁱ⁾
      ⋮
wₙ := wₙ − α · 1/m ∑i=1m (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾) xₙ⁽ⁱ⁾
b   := b   − α · 1/m ∑i=1m (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾)
All updates are computed simultaneously using the old values of all parameters — same rule as Module 2.

The Pattern Is Always the Same

Notice the pattern: the update for wⱼ is exactly the Module 2 update for w, but with xⱼ⁽ⁱ⁾ (feature j of example i) in place of the single x⁽ⁱ⁾. And the update for b is unchanged. We can write this compactly as:

Update for wⱼ (j = 1 to n) wⱼ := wⱼ − α · ∂J/∂wⱼ

where ∂J/∂wⱼ = (1/m) Σ (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾) xⱼ⁽ⁱ⁾
Update for b (unchanged from Module 2) b := b − α · ∂J/∂b

where ∂J/∂b = (1/m) Σ (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾)
🔑 What changed from Module 2?

Only one thing: instead of one w, we now update n weights, each using its own feature xⱼ in the gradient. The b update is identical to Module 2. The simultaneous update rule still applies — you must compute all gradients using old values first, then assign all new values.

✅ Great news

The jump from n = 1 to n features is just adding more w updates. Everything else — the cost function formula, the b update, the simultaneous update rule, the learning rate α, convergence detection — is exactly the same as Module 2.

Section 04

Feature Scaling — Making Gradient Descent Work Fast

When features have very different ranges — say x₁ ∈ [1, 3] (floors) and x₁ ∈ [300, 3000] (sq ft) — gradient descent has a serious problem. Let's understand why, and how to fix it.

The Problem — Elongated Cost Contours

💡 Intuition — Why elongated contours are bad

When x₂ has a much larger range than x₁, a small change in w₂ causes a big change in J, but a small change in w₁ causes almost no change. Gradient descent must use a tiny α to avoid overshooting on the w₂ axis — which makes it crawl along the w₁ axis. The solution: put all features on the same scale first.

Method 1 — Simple Rescaling (Divide by Max)

The simplest approach: divide each feature by its maximum value so it falls in the range [0, 1]:

Simple rescaling
x̃ⱼ = xⱼ / max(xⱼ)
After this, all features lie in [0, 1]. Quick and easy, but sensitive to outliers.

Method 2 — Mean Normalisation

Andrew Ng's preferred method: shift each feature to be centred around zero, then divide by the range:

Mean Normalisation
x̃ⱼ = (xⱼ − μⱼ) / (max(xⱼ) − min(xⱼ))
μⱼ is the mean of feature j. After this, features are roughly centred at 0 and lie in [−1, 1].

Method 3 — Z-score Standardisation

The most statistically principled method: shift to zero mean and divide by standard deviation σⱼ:

Z-score Standardisation
x̃ⱼ = (xⱼ − μⱼ) / σⱼ
σⱼ is the standard deviation of feature j. Result: every feature has mean 0 and standard deviation 1.

A Concrete Example

Let's apply mean normalisation to our house-price features. Given: size ∈ [300, 2000], bedrooms ∈ [0, 5], mean size = 1000, mean bedrooms = 2.5:

Feature Range μ (mean) Raw value (example) After mean norm.
x₁ (size, sq ft) 300 – 200010002104 (2104 − 1000) / 1700 ≈ 0.65
x₂ (bedrooms) 0 – 52.55 (5 − 2.5) / 5 = 0.50

Both features are now in a similar range — gradient descent will behave much better.

⚠️ Critical Rule — Scale using training data only

Compute μⱼ, σⱼ, min, max using only the training set. Then apply the same scaling to the test/validation set and new predictions. Never compute scaling statistics from test data — that would be "data leakage".

When to scale

Aim for features roughly in the range [−1, 1] or [−3, 3]. If any feature has a much larger or smaller range than the others, scale it. If features are already on similar scales (e.g. all probabilities in [0,1]), scaling has little effect and is not strictly necessary.

Section 05

Choosing the Learning Rate α

One of the most practical skills in machine learning is knowing how to set α. Set it wrong and gradient descent will either crawl or diverge. Here is Andrew Ng's systematic approach.

The Learning Curve is Your Diagnostic Tool

Always plot J vs number of iterations after running gradient descent. The shape of this curve tells you everything:

The Step-by-Step Procedure

1

Start with a very small α (e.g. 0.001)

Run gradient descent for ~100 iterations and plot J. It should be going down. If even 0.001 makes J go up, check your code — there is likely a bug (a common one is forgetting to simultaneously update w and b).

2

Increase α by ~3× each time

Try 0.001 → 0.003 → 0.01 → 0.03 → 0.1 → 0.3 → 1. For each, plot J vs iterations. You're looking for the largest α where J still monotonically decreases.

3

Find the boundary — then back off slightly

Say 0.1 works but 0.3 diverges. Pick something in between, like 0.1 or 0.2. There's no single right answer — you want fast convergence without instability.

4

After feature scaling, revisit α

Feature scaling (Section 04) often lets you use a much larger α. Always re-tune α after scaling your features — the optimal value can change significantly.

🔑 The golden rule

With a small enough α, gradient descent will always decrease J on every iteration for linear regression (because J is convex). If J ever increases, it is always either a too-large α or a bug in your code. This is a useful debugging principle.

Section 06

Polynomial Regression — When a Line Isn't Enough

Why Linear Might Not Always Be the Best Fit

Linear regression fits a straight line (or hyperplane) through the data. But what if the true relationship is curved? Consider house prices vs. size:

💡 Andrew Ng's Key Observation

Andrew Ng says: "The choice of features is up to you." You can take your original feature x (like house size) and engineer new features from it — like x², x³, or √x. Once you create these new features, you still run the same linear regression algorithm. The model is linear in the parameters w, even if it is nonlinear in the original feature x.

How Polynomial Regression Works

The trick is simple: create new features by raising x to higher powers. Then treat each power as a separate feature and apply multivariate linear regression.

1

Start with your original feature x

Say x = house size (sq ft). We observe the data looks curved, so a straight line won't do.

2

Create engineered features

Define x₁ = x, x₂ = x², x₃ = x³ (or as many powers as you think you need). These are just new columns in your feature matrix.

x₁ = x  (size)    x₂ = x²  (size²)    x₃ = x³  (size³)
3

Write the polynomial model as a linear regression

Now the model looks exactly like multivariate linear regression with n = 3 features:

ŷ = w₁x₁ + w₂x₂ + w₃x₃ + b = w₁x + w₂x² + w₃x³ + b
4

Run the same gradient descent algorithm

Gradient descent, the cost function J, and the update rules are completely unchanged. The algorithm doesn't know or care that x₂ = x₁². It just sees three features and finds the best w₁, w₂, w₃, b.

General degree-d polynomial model
ŷ = w₁x + w₂x² + w₃x³ + … + wdxd + b
This is still a linear regression problem — linear in the weights w. Same J, same gradient descent, same update rules.

Alternative: The Square Root Feature

Sometimes a curve levels off as x grows — like house prices growing quickly for small houses but tapering off for very large ones. A square root feature can capture this:

Square root model
ŷ = w₁x + w₂√x + b
√x grows quickly at first then slows down — perfect for diminishing returns. Define x₂ = √x and you have a standard 2-feature linear regression.
⚠️ Feature Scaling is Critical for Polynomial Features

If x ranges from 1 to 1000, then x² ranges from 1 to 1,000,000 and x³ from 1 to 1,000,000,000! These wildly different scales make gradient descent behave terribly. Always apply feature scaling (Section 04) when using polynomial features.

How Do You Choose Which Features to Use?

Andrew Ng's practical advice:

  • Plot your data first. Look at scatter plots of each feature vs y. Does the relationship look linear, curved, levelling off?
  • Start simple. Try linear first. If J is still large after convergence, try adding quadratic or sqrt features.
  • Use learning curves. If adding a new feature consistently reduces J on both training and validation sets, it's useful.
  • Domain knowledge helps. In physics, you might know that drag force ∝ velocity² — so you'd add a v² feature deliberately.

Practice Questions

Think & Answer

Work through these questions on your own before revealing the hints. They cover multiple features, feature scaling, and polynomial regression — exactly the kinds of questions you'd see in an exam on this module.

Easy Medium Hard
Part A — Conceptual & Intuition
Q1 In multiple linear regression, what does each weight wⱼ represent? How is it different from the single-feature case?
Easy Concept
▾
Sample Answer

Each weight wⱼ tells you how much the prediction changes when feature xⱼ increases by one unit, while all other features are held constant. In the single-feature case there is only one w, so there is no need for subscripts. With multiple features, there is a separate weight for every feature, and the model is a hyperplane in n-dimensional space instead of a line in 2D.

Q2 Why is feature scaling important for gradient descent? What can go wrong if you skip it?
Easy Concept
▾
Sample Answer

When features have very different ranges (e.g. house size in thousands vs number of bedrooms from 1–5), the cost surface becomes a narrow, elongated ellipse. Gradient descent then oscillates back and forth along the steep axis and makes slow progress. Feature scaling makes the cost surface more circular, so gradient descent moves directly toward the minimum and converges much faster.

Q3 What is the difference between mean normalisation and Z-score standardisation? When would you prefer one over the other?
Medium Concept
▾
Sample Answer

Mean normalisation rescales values to roughly [−1, 1] using: x' = (x − μ) / (max − min). Z-score standardisation rescales so that the feature has mean 0 and standard deviation 1: x' = (x − μ) / σ.

Z-score is generally preferred because it handles outliers more robustly (the std deviation captures spread, not just max–min). Mean normalisation is simpler and works well when data has no extreme outliers.

Q4 Why does the cost function stay the same (MSE) when we move from one feature to many features?
Easy Concept
▾
Sample Answer

The cost function measures the difference between the predicted output ŷ and the true output y, regardless of how ŷ was computed. Whether ŷ comes from one feature or fifty, it is still a single number, so the squared error (ŷ − y)² is still a single number. The only thing that changes is how ŷ is calculated — it now uses a dot product of a vector w with x, but the cost formula itself is unchanged.

Q5 What is polynomial regression? Is it still considered "linear" regression? Explain why or why not.
Medium Concept
▾
Sample Answer

Polynomial regression fits a curved relationship by adding higher-order terms like x², x³ as new features. For example, treating x² as a separate feature x₂ = x² turns a curve-fitting problem into a linear regression problem in the expanded feature space.

It is still called linear regression because the model is linear in the parameters (the weights w). Even though the input features are non-linear transformations of x, the model is still ŷ = w₁x + w₂x² + b, which is linear in w₁ and w₂.

Part B — Calculation & Application
Q6 A house has features x = [1400, 3, 2, 10] (size in sq ft, bedrooms, bathrooms, age in years) and the model has w = [0.15, 4, 6, −0.1] and b = 50. What is the predicted price ŷ?
Easy Math
▾
Sample Answer

Compute the dot product wᵀx + b:

ŷ = (0.15×1400) + (4×3) + (6×2) + (−0.1×10) + 50 ŷ = 210 + 12 + 12 − 1 + 50 = 283

So the predicted price is $283k. This is a straightforward dot product — always work through each feature one term at a time.

Q7 A feature x has values [100, 200, 300]. Apply Z-score standardisation and write the scaled values. (μ = 200, σ = 81.6)
Easy Math
▾
Sample Answer

Apply x' = (x − μ) / σ for each value:

x'₁ = (100 − 200) / 81.6 ≈ −1.22 x'₂ = (200 − 200) / 81.6 = 0.00 x'₃ = (300 − 200) / 81.6 ≈ +1.22

After scaling, the mean is 0 and values are symmetric around it. This is the expected pattern for Z-score standardisation.

Q8 Write the gradient descent update rule for weight w₂ in a model with 3 features. Identify every term and explain what it does.
Medium Math
▾
Sample Answer
w₂ := w₂ − α · (1/m) Σᵢ (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾) · x₂⁽ⁱ⁾

α = learning rate (step size). 1/m = average over all m training examples. (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾) = prediction error for example i. x₂⁽ⁱ⁾ = the value of feature 2 for example i. The whole sum is the partial derivative of the cost with respect to w₂, telling us which direction to move w₂ to reduce the cost.

Q9 You want to model house price as a function of size using a cubic polynomial. Write the model equation and list the features you would feed to a standard linear regression algorithm.
Medium Math
▾
Sample Answer

Define new features: x₁ = size, x₂ = size², x₃ = size³. The model equation is:

ŷ = w₁x₁ + w₂x₂ + w₃x₃ + b

You then feed the three-column feature matrix [x₁, x₂, x₃] to the standard multiple linear regression algorithm. Feature scaling is especially important here because x³ can be astronomically large compared to x₁.

Q10 A model with 2 features has w = [2, −3] and b = 1. A single training example is x⁽¹⁾ = [1, 2] and y⁽¹⁾ = 0. Assuming m = 1, compute one gradient descent update for w₁, w₂, and b using α = 0.1.
Hard Math
▾
Sample Answer

Step 1 — Forward pass:

ŷ = w₁x₁ + w₂x₂ + b = 2(1) + (−3)(2) + 1 = 2 − 6 + 1 = −3

Step 2 — Errors:

error = ŷ − y = −3 − 0 = −3

Step 3 — Gradients (m = 1):

∂J/∂w₁ = error · x₁ = −3 · 1 = −3 ∂J/∂w₂ = error · x₂ = −3 · 2 = −6 ∂J/∂b = error = −3

Step 4 — Updates:

w₁ := 2 − 0.1(−3) = 2.3 w₂ := −3 − 0.1(−6) = −2.4 b := 1 − 0.1(−3) = 1.3

Notice all three updates must be computed using the original values before any of them are applied (simultaneous update rule).
