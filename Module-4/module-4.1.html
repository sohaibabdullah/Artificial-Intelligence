Neural Networks
— Motivation & Forward Pass

We have so far used logistic regression (Module 3) to classify. But what if the data is far too complex for any hand-crafted combination of features to separate? Neural networks learn their own features automatically — and they do it by stacking the very same building block we already know: the logistic unit.

Section 01

Why Do We Need Neural Networks?

In Module 2.1 we added polynomial features like x₁², x₁x₂, x₂² to linear regression to handle curves. In Module 3 we used the same trick for logistic regression to get non-linear decision boundaries. This works well for small problems. But it breaks down quickly.

The Feature Explosion Problem

Suppose you are building a classifier to detect whether an image contains a car. A modest image has 50×50 pixels = 2,500 pixels. If you include all quadratic feature pairs (x₁x₂, x₁x₃, …), you end up with roughly 3,000,000 features. This is computationally impossible to train and extremely prone to overfitting.

🔑 The Neural Network Insight

Instead of we manually engineering features, what if the algorithm could learn its own features automatically from raw data? That is exactly what a neural network does. It learns a hierarchy of increasingly abstract features — layer by layer — without us having to specify them. And it does this by stacking the same basic building block we already know from Module 3: the logistic unit.

A Bridge From What You Already Know

Remember Module 3's logistic regression model: take inputs x, compute z = wᵀx + b, then apply the sigmoid to get ŷ = g(z). A neural network is simply many of these logistic units connected together in layers. The output of one layer becomes the input to the next. That's it — the concept is an elegant extension of what you already understand.

Section 02

The Artificial Neuron — The Atom of a Neural Network

Before building a full network, let's understand the single unit that everything is built from. We can call it a neuron or a unit. It is a direct extension of logistic regression.

Biological Inspiration (Brief)

A biological neuron receives signals through dendrites, combines them in the cell body, and fires an output through the axon if the combined signal is strong enough. The artificial neuron is a loose mathematical analogy: it receives several numbers as inputs, takes a weighted sum, adds a bias, and passes the result through an activation function to produce one output number.

✅ Connection to Module 3

In Module 3, the output of logistic regression was ŷ = g(wᵀx + b). A single neuron is exactly this — but we now call the output a instead of ŷ. We use a (short for activation) because in a network this output immediately becomes the input to the next layer — it is not yet the final prediction. The notation change reflects this role: a is an intermediate activation, ŷ is reserved for the final network output.

The Activation Function

The sigmoid g(z) = 1/(1+e⁻ᶻ) is one choice of activation function. Its job is to introduce non-linearity. Without an activation function, stacking many linear layers would still just give a single linear transformation — we'd gain nothing. The activation function is what gives neural networks their power to represent complex, non-linear patterns.

In this module we use the sigmoid (consistent with Module 3). Module 4.2 will introduce other popular choices like ReLU.

Section 03

Layers & Architecture — Stacking Neurons

One neuron gives us logistic regression. The power of neural networks comes from organising many neurons into layers and stacking those layers one after another. Let's build this up gradually.

A Network with One Hidden Layer

The simplest useful neural network has three groups of neurons: an input layer, one hidden layer, and an output layer. Let's see what each one does and why.

What Does Each Layer Do?

0

Input Layer (Layer 0)

This is simply the raw input features x₁, x₂, …, xₙ from the training data. It does no computation — it just passes values in. By convention this is called layer 0 and is not counted when we say a network has "L layers".

1

Hidden Layer (Layer 1, 2, …, L−1)

The neurons here are called "hidden" because we don't observe their values in the training data — we only observe x (input) and y (output). The hidden layer learns intermediate features automatically. For example, in an image network an early hidden layer might learn to detect edges; a later hidden layer might learn to combine edges into shapes. We never specify this — the network learns it.

2

Output Layer (Layer L)

The final layer produces the network's prediction ŷ. For binary classification (like Module 3), the output layer has a single neuron with sigmoid activation, giving ŷ ∈ (0,1) — the probability that y=1. For multi-class problems we use more output neurons (covered later).

⚠️ Why is the middle layer called "hidden"?

Because we never see its values in the dataset. The dataset only contains the input features x and the true label y. What the hidden neurons compute is entirely internal to the network — hence "hidden". This is different from a missing feature; it's a feature the network invents for itself during training.

Section 04

Notation — Reading Neural Network Equations

We will use a superscript-in-square-bracket notation for neural networks. It looks intimidating at first, but it is logical once you know the pattern. Every symbol has at most two indices, each with a clear meaning.

📌 The Two Index Convention

Superscript [l] in square brackets always means layer number. Subscript j (or i) always means unit number within that layer. That's it.

🔑 Key Link to Module 3 Notation

In Module 3, a single logistic unit had one weight vector w and one bias b. In a neural network, each layer has a weight matrix W[l] (one row per neuron in that layer) and a bias vector b[l]. The parameters of the whole network are {W[1], b[1], W[2], b[2], …, W[L], b[L]}. Everything else — the sigmoid, the cost function, gradient descent — follows the same principles as Module 3.

Section 05

Forward Pass — Computing One Layer at a Time

The forward pass is the process of computing the network's output ŷ from a given input x. It works by moving from left to right through the network — layer by layer — computing each layer's activations from the previous layer's activations. Let's trace through one layer first, step by step.

Computing One Neuron in the Hidden Layer

Take unit 1 of layer 1 (the top hidden neuron in Figure 3). It receives x₁, x₂, x₃ as input, has its own weights w₁[1], w₂[1], w₃[1] and bias b₁[1]. It computes in two steps:

1

Linear combination — exactly like Module 3

Compute the weighted sum of inputs and add the bias:

z₁[1] = w₁₁[1]·x₁ + w₁₂[1]·x₂ + w₁₃[1]·x₃ + b₁[1]
where w₁ⱼ[1] means: weight from input j to unit 1 of layer 1
2

Apply activation function — again same as Module 3

Pass z through the sigmoid to get this neuron's output activation:

a₁[1] = g(z₁[1]) = 1 / (1 + e−z₁[1])

Now repeat the exact same two steps for units 2, 3, 4 — each has its own weights and bias, but the computation structure is identical:

💡 Each Neuron Learns a Different Feature

Even though all four neurons receive the same inputs x₁, x₂, x₃, they have different weights. So each neuron learns to detect a different pattern in the input. Neuron 1 might learn "is feature 1 large?", neuron 2 might learn "is feature 1 small but feature 2 large?", and so on. The network discovers which combinations matter — you don't have to tell it.

Section 06

Forward Pass — Through the Full Network

We've seen how layer 1 computes its activations from the inputs. Now let's trace the full forward pass: from x all the way to ŷ, passing through every layer.

Layer 2 — The Output Layer

The output layer (layer 2 in Figure 3) has just one neuron. Its input is not x — it's the 4-dimensional activation vector a[1] from layer 1. The computation is the same two-step process:

Output layer computation
z₁[2] = w₁₁[2]·a₁[1] + w₁₂[2]·a₂[1] + w₁₃[2]·a₃[1] + w₁₄[2]·a₄[1] + b₁[2] ŷ = a₁[2] = g(z₁[2])
Same two-step formula. The only change is that the inputs are now a₁⁽¹⁾…a₄⁽¹⁾ instead of x₁…x₃. The output of the last layer is the final prediction ŷ.

Tracing the Complete Forward Pass End-to-End

✅ The Pattern — Each Layer Follows the Same Two Steps

Every single layer l in the network computes exactly the same pair of equations. Only the layer index changes:

Forward pass equations for layer l — the general rule
z[l] = W[l] · a[l−1] + b[l] a[l] = g(z[l])
Start with a[0] = x. Repeat for l = 1, 2, …, L. The final result a[L] = ŷ.

To produce ŷ from a new input x, run this two-equation pair L times — once per layer. That is the entire forward pass algorithm.

Section 07

Vectorised Forward Pass — From One Example to All m at Once

So far the forward pass equations describe computing ŷ for a single training example x. In practice we always want to compute ŷ for all m training examples simultaneously. We do this by stacking all examples into a matrix — exactly as Module 2.1 did with the feature matrix X.

Stacking Examples Into a Matrix

Instead of one column vector x ∈ ℝn×1, we form the matrix X by placing all m examples side by side as columns:

Input matrix X — all m examples at once
X = [x(1) | x(2) | … | x(m)]   ∈ ℝn × m
Each column is one training example. X has n rows (features) and m columns (examples). This is a transpose of how Module 2 stacked examples — here columns are examples, not rows. Andrew Ng uses this convention for neural networks.
📌 Notation difference from Module 2

In Module 2, the feature matrix had shape (m × n) with each row being one example. In neural networks, Andrew Ng uses shape (n × m) with each column being one example. This is a convention choice — it makes the layer equations cleaner. Either convention is valid; just be consistent.

The Vectorised Forward Pass Equations

With X ∈ ℝn×m, the forward pass for all m examples simultaneously uses exactly the same two equations per layer — just with matrices instead of vectors:

Vectorised forward pass for layer l — all m examples at once
Z[l] = W[l] · A[l−1] + b[l] A[l] = g(Z[l])
Capital letters denote matrices over all m examples. Z[l] and A[l] both have shape (n[l] × m). Start with A[0] = X. The final A[L] has shape (1 × m) — one prediction per example.

✅ Why vectorise?

Computing the forward pass for all m examples simultaneously using matrix multiplication is vastly faster than looping over examples one at a time — for the same reason we used NumPy's @ operator in Implementation 1. Modern hardware (GPUs) can multiply large matrices extremely fast. This is why neural networks, despite having millions of parameters, can be trained in reasonable time.
